5 - When to use the third rule

If we cast our minds back to the original explination of collapsing margins..

"When two block level elements of the same block formatting context adjoin vertically"

an explanation of block formatting contexts can be found here.

so the reason our margins collapse in the previous example is because the frame (div3) and the last child element (div1) both have the same block formatting context.
The key to understanding the third rule is to realise that this will not always be the case

Grids

A large part of the layout of a webpage will be defined by a grid system.

Whether you are using a ready made grid system Like bootstrap or are creating your own, they generally rely on floating DIV elements to create the columns.
If you then place standard block elements inside that floated DIV they will be a different block formatting context

lets try that out by adding float to div3 from the previous example 

    .div3 {
      float:left;
      width: 100%;
    }

et voil√†
we can now see the bottom margin from the last child element and the bottom margin from our container (div3)

So for these circumstances (the first level inside the grid) we don't have to do anything and our margins will render as expected
