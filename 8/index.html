Don't forget that as ever the borders are calculated in ems. this means if the font size changes
you need to recalculate the elements bottom padding and border width in relation to the new font size

Consider the two examples below

style.scss 
        .left {
          color: blue;
          @include adjust-font-size-to(18px);
          @include trailing-border(4px)
        }

Compiles to

style.css 
        .left {
          color: blue;
          font-size: 1.5em;
          line-height: 2em;
          border-bottom-style: solid;
          border-bottom-width: 0.33333em;
          padding-bottom: 1.16667em;
        }
Test line for .left class
style.scss 
        .right {
          color: green;
          @include adjust-font-size-to(18px);
          @include trailing-border(4px,1,18px)
        }

Compiles to

style.css 
        .right {
          color: green;
          font-size: 1.5em;
          line-height: 2em;
          border-bottom-style: solid;
          border-bottom-width: 0.22222em;
          padding-bottom: 0.77778em;
        }
Test line for .right class


On the left side the border and padding is incorrectly calculated based on the base font-size. This results
in too much padding and too thick a border.
on the right side we have told trailing-border() our font-size, resulting in the right sizes.

trailing-border variables

trailing-border($width, $lines, $font-size, $border-style)

by changing the $lines variable we can set how many lines the bottom padding takes up 

    .three-lined-border {
      color: red;
      @include trailing-border(4px, 3)
    }
