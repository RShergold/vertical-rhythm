Vertical Rhythm

3 - Add rhythms to margins

the first thing we are going to do is remove all of the default margins and padding from all block level elements we are going to use

style.scss 
    body, div, h1, h2, h3, p, pre {
      margin: 0;
      padding: 0;
    }

adding the rhythm

Then we add rhythm to each block element

style.scss 
    body {
      margin: rhythm(1);
    }

    p {
      margin-bottom: rhythm(1);
    }

which compiles to

style.css 
    body {
      margin: 1.5em;
    }

    p {
      margin-bottom: 1.5em;
    }

Nicely aligned

every line now fits within the bounds of our 18px guide lines. The only elements that have a margin are BODY and P.
Those margins take up exactly one line.

However, our headers seem a bit wrong. They need some spacing... What's going on there?

The headers have the line height set in 

    html {
      line-height: 1.5em
    }

as this line-height is calculated in relation to the font-size of HTML it results in a line-height of 18px.
that's exactly what we need to fit it nicely in our guide lines but is not the correct line height for the headers as
they are a larger font size.
