Vertical Rhythm

6 - Mix font size and margins

So we know how to add margins to things with rhythm(1), and how to change font sizes with adjust-font-size-to().
What happens when you combine the two?

style.scss 
        .left {
          color: blue;
          @include adjust-font-size-to(18px);
          margin-bottom: rhythm(1)
        }

Compiles to

style.css 
        .left {
          color: blue;
          font-size: 1.5em;
          line-height: 2em;
          margin-bottom: 1.5em;
        }

Test line for .left class

This side breaks the rhythm. If we inspect our left test line, you can see that the bottom margin covers more than one line.

This is because rhythm(1) outputs an em calculated against the base font size.
Because we changed the font-size the em's are now calculated against the new font-size rather than the base.

style.scss 
        .right {
          color: green;
          @include adjust-font-size-to(18px);
          margin-bottom: rhythm(1, 18px)
        }

Compiles to

style.css 
        .right {
          color: green;
          font-size: 1.5em;
          line-height: 2em;
          margin-bottom: 1em;
        }

Test line for .right class

we can fix this by calculating our rhythm against the new font-size by simply supplying rhythm() with a second argument equal to the new font size.






This causes issues if you are using a class to change the font size of an element and a different class to add padding. When those classes are applied to the same element, this approach will always break the rhythm... unless you create classes for every combination of margin and font size.
